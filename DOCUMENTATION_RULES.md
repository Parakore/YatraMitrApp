# 🧭 Documentation & Maintenance Rules (YatraMitra)

You are responsible for maintaining consistency across:
- Codebase
- Documentation files
- Comments
- AI-generated output

## 📂 Required Documents
- `README.md`
- `ARCHITECTURE.md`
- `RULES.md`
- `AI_GUIDELINES.md`
- `CONTRIBUTING.md`
- `GOVERNANCE.md`

## 🔄 Update Frequency
Documentation MUST be updated when:
- Architecture changes.
- Coding rules change.
- Setup processes change.

## 💬 Commenting Rules
- Add comments for complex logic and business rules.
- Avoid commenting obvious code.
- Use `///` for class and function descriptions.

## 🧩 Feature Documentation
Each feature should have its own `README.md` within its directory explaining:
- Purpose
- Flow
- Dependencies
- Offline behavior
